Dear Reviewer,

Gistwarden is a secure password, passkey (FIDO2), and TOTP manager that encrypts vault data locally and syncs it directly to the user's private GitHub Gists.

Official GitHub Repository: https://github.com/uongsuadaubung/gistwarden

This extension contains bundled code generated by esbuild and compiled SCSS. The complete source code package is provided herewith.

Here are the step-by-step instructions to build the extension from this source code:

1. Environment Requirements:
   - Ensure Deno is installed on your system (v2.x). If not installed, you can download it from https://deno.land/

2. Extraction:
   - Extract the submitted source code ZIP file into a workspace directory.

3. Dependencies and Build:
   - Open your terminal and navigate to the extracted source directory.
   - Run the build task using the following command:
     deno task build

4. Output Verification:
   - After the build successfully completes, the compiled extension packages will be generated inside the `dist` directory:
     * For Chrome: The compiled files are inside the `dist/chrome` directory, and the packed package is created at `dist/chrome.zip`.
     * For Firefox: The compiled files are inside the `dist/firefox` directory, and the packed package is created at `dist/firefox.zip`.
   - The compiled directory (`dist/chrome` or `dist/firefox`) represents a 100% exact copy of the submitted extension package.

Testing Notes:
- To test the full functionality (including sync), you will need a GitHub account and a Personal Access Token (PAT) with only the `gist` permission scoped.
- However, you can also test the core offline functionality (creating and managing passwords locally) by simply setting a Master Password without configuring the GitHub sync.
